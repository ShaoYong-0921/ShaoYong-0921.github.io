## 事件與教訓

1. **`.git` 洩漏進同步資料庫**：livesync-bridge 的 storage peer **沒有排除規則**，vault 內的 `.git/` 被整包同步進 CouchDB（44 份文件）。解法：git 目錄外移至 `~/.vault-git`（設計文件已補記），洩漏文件已批次刪除。教訓：接第三方工具前先讀它的邊界行為。
2. **容器權限三連戰**：橋容器預設 deno uid 寫不進 vault（讀得到寫不進的不對稱是線索）→ 改 `user: 1000` 後 DENO_DIR 不可寫（同步檢查點存 localStorage）→ 快取卷又是 root 所有。最終解：uid 1000 + 可寫的 `HOME`/`DENO_DIR` 掛載。教訓：容器掛主機目錄時，**uid 對齊**是第一件要想的事。
3. **派負載 10、71°C**：桌面上遺留的 Chromium 把四核心吃滿，過熱降頻造成「當機感」。風險表「實測 70°C」應驗。行動項：M3 加固提前（關 lightdm、散熱）。
4. **CouchDB 密碼在終端機截圖中外洩**（貼進了對話）：僅 Tailscale 私網可達、風險有限，但違反機密紀律。**待辦：輪換密碼**（.env + bridge config + Obsidian 欄位 + 容器重建）。
5. **`pkill -f chromium` 殺掉自己的 SSH**：`-f` 比對整條命令列，而指令本身含關鍵字。用 `pkill 程序名` 或 `pkill -f "[c]hromium"`。
6. **dry-run ≠ 正式執行**：`--dry-run` 只印計畫不寫檔，後續 git add 自然找不到東西。兩段式（先 plan 再 apply）是好習慣，但別忘了第二段。
