## 網域決策記錄

- 候選：Cloudmax `.tw`（NT$550）、Cloudflare `.uk`（NT$150）、`shao-yong.com`、`shaoyong.dev`（$12.20/年）
- 淘汰 `.uk`：Nominet 要求英國送達地址（資格風險）+ ccTLD 會被搜尋引擎地理定向到英國（受眾在台灣，自傷 SEO）
- 淘汰 `shao-yong.com`：連字號口頭難傳達、易打錯、觀感差
- 選 `shaoyong.dev`：無連字號、工程師慣例 TLD、Google registry 強制 HTTPS、Cloudflare 成本價續約不漲、與既有 Cloudflare DNS 規劃一體

## 綁定流程（可複用的 SOP）

1. Cloudflare DNS 兩條 CNAME：`@` 和 `www` → `shaoyong-0921.github.io`，**都必須灰色雲（DNS only）**——橘色雲（proxy）會擋 GitHub 的憑證驗證
2. GitHub repo Settings → Pages → Custom domain 填 `shaoyong.dev`
3. GitHub 偵測 DNS 變更後重簽憑證，進度分三階段（Requested → Validation → Issued），約 15 分鐘～1 小時；期間主網域照常服務
4. 憑證好後勾 **Enforce HTTPS**
5. `astro.config.mjs` 的 `site` 改為新網域，push 重部署（影響 sitemap/RSS 網址）
6. 驗證四路徑：apex 200、www 301→apex、http 301→https、舊 github.io 301→新網域

## 問題與解法

1. **GitHub 報 www InvalidDNSError**：綁自訂網域時 GitHub 會連 www 變體一起檢查簽憑證，DNS 必須補 www 記錄，缺了憑證就發不下來。
2. **加 www 後憑證等很久**：DNS 變更觸發整張憑證重新申請（涵蓋 apex+www），流程重跑，屬正常。不要按 Remove 重來，會越弄越久。
