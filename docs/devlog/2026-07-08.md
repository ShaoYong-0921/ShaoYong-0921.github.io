## 問題與解法

1. **忘記派的密碼** → code-server 的終端機不需要 Linux 密碼，從那裡把 SSH 公鑰放進 `authorized_keys`，之後全走金鑰。教訓：帳號是 `homepi` 不是 `pi`。
2. **誤跑 `pnpm dlx @astrojs/upgrade`**，Astro 5 → 7，主題直接壞掉。決定不降版、把主題移植到 Astro 7，共修五處：
   - 裝 `@astrojs/markdown-remark`（v7 換了預設 Markdown 處理器）
   - content collections 改新 API：`src/content.config.ts` + glob loader、`entry.slug`→`entry.id`、`entry.render()`→`render(entry)`
   - expressive-code 升 0.44（0.41 不支援 Astro 7）
   - `markdown.css` 兩處跨檔 `@apply` 自訂 class 展開（Vite 8 逐檔處理 CSS）；`cssMinify: "esbuild"`（lightningcss 不吃殘留巢狀）
   - ImageWrapper：glob key 改路徑結尾比對；封面 basePath 改用 `entry.filePath`（新 API 的 id 無副檔名，`getDir(id)` 對子資料夾文章會算錯）
3. **封面圖消失**：就是上面 basePath 的問題，子資料夾文章（`guide/index.md`）才會踩到。教訓：build 綠燈不代表沒問題，「Image file not found」只是警告不擋 build——驗證要看輸出的 HTML。
4. **GitHub 內建 Jekyll 建置紅燈**：Pages Source 要設「GitHub Actions」而不是「Deploy from a branch」。
5. **未提交的修改被覆蓋**：banner 設定改在工作目錄沒 commit，被後續的檔案覆蓋洗掉。教訓：**改了就先 commit，wip 也commit**；覆蓋檔案前先 `git status`。

## 學到的概念

- Astro = build 時把 Markdown + 模板組成靜態 HTML，訪客拿到的預設零 JS；互動處用 island（`client:only` 的 Svelte 元件）
- content collections ≈ 把 Markdown 當資料庫：collection=table、frontmatter schema=zod 驗證、`getCollection()`=SELECT
- Tailwind 不是模板，是 build 時掃描 class 生成 CSS 的零件庫；Fuwari 的外觀是主題作者用它拼的
- CI 門檻：workflow 之間預設平行，要「紅燈不部署」得用 `needs:` 串依賴
